AI Agents, Productivity, and Higher-Order Thinking: Early Evidence From Software Development

Suproteem K. Sarkar • SSRN First Look (Nov 2025)

View paper ↗

Cursor’s release of a coding agent created a natural experiment across 32 organizations. Once the agent became the default code-generation mode, weekly merges rose 39% and lines edited rose 49%, with no measurable uptick in revert rates and a 2.4 percentage-point drop in bugfix share. A separate user-level panel shows that each standard deviation (~6.6 years) of additional work experience lifts agent accept rates by 5–6% and that plan-first prompts correlate with higher-quality delegates—evidence that agents are shifting work toward abstraction, clarity, and evaluation.

Plain-language explainer

Instead of typing every line yourself, senior engineers now describe the change, ask the agent to draft a plan, and only step in to edit or approve. The study shows that teams who make agents the default “first drafter” get more code merged without introducing more breakages—provided humans stay in the loop to define intent and review outputs.

🧭

Executive quick take

Treat coding agents like a productivity lever, not a toy. Cursor’s diff-in-diff shows the lift only materialized when the agent was the default path and organizations had enough experienced reviewers to steer it. Track semantic work (plans, explanations) alongside merges, because the cognitive bottleneck moves upstream to specification and downstream to evaluation.

  • Defaulting the agent matters: Optional access yielded +26% merges, but making agents the default pushed output to +39% without harming revert rates.
  • Experience gradient is real: Every +6.6 years of tenure correlated with ~6% higher accept rates, while autocompletion showed the opposite (junior-skew) pattern.
  • Work patterns shift to semantics: 39% of first agent messages are non-implementation (plan, explain, diagnose) and plan-first users accept edits more often, signaling a need for prompt design and review rituals.

💼 Business relevance

  • Engineering & platform leads: Quantify merges, lines edited, and revert/break-fix rates before and after agent rollout. Expect ~30–40% uplift if the agent is default and reviewers stay in the loop.
  • Product/design teams: Non-engineering roles submitted 50–75 agent prompts per week; they can prototype UIs or logic themselves, freeing engineers for integration work.
  • People ops & enablement: Training should emphasize plan-first messaging, review checklists, and mental models of agent capability—experienced workers outperform because they invest here.

Derivative example

A 180-engineer fintech firm makes Cursor’s agent the default editor mode. Week 0–2: baseline telemetry (merges, reverts, plan-first share). Week 3: roll out default agent, require “plan” prefix for risky changes, add reviewers for final approval. Week 8: merges up 34%, bugfix share down 2 pp, plan-first share up from 12% to 24%, enabling a 4-person reduction in backlog triage. Outcome: +$2.3M/year productivity, no increase in Sev1s.

Difference-in-differences natural experiment

Cursor released its agent on Nov 24, 2024 and flipped the default on Feb 19, 2025. The paper compares 24 “eligible” orgs already on Cursor against eight “baseline” orgs that had not adopted yet, controlling for week and org fixed effects. This isolates the productivity change attributable to the agent release rather than seasonal output swings.

Experience-driven delegation skill

Analyzing 1,000 users’ agent logs shows the least-experienced decile sends many prompts but accepts fewer edits. Each standard deviation of tenure increases accept rates by ~6%, even after controlling for role, sector, and seniority. Plan-first prompts (13.6% of messages) are more common among experienced workers and predict higher accept rates—delegation skill is the differentiator.

Key insight

Agentic coding tools boost throughput once they become the default workflow: +39% merges and +49% lines edited with no short-run quality penalty, but only when organizations treat specification and review as first-class work.

Method

A dynamic difference-in-differences model with org and week fixed effects estimates the causal impact of the agent release. Complementary cross-sectional regressions link message intent, usage frequency, and experience to accept rates. Message classification models tag first prompts as implement/plan/explain/error to quantify semantic shifts.

Implication

Rollouts should pair default-on agents with enablement that builds planning and evaluation muscle. Monitor plan-first usage, accept rates, and quality proxies to ensure uplift stems from better delegation—not reckless auto-merges.

🧪 Evidence

  • Default-on uplift: Eligible orgs saw +39% weekly merges and +49% thousands of lines edited after the agent became default; revert rates stayed flat (−0.08 pp) while bugfix share fell −2.40 pp.
  • Cross-functional adoption signal: Designers submit 74.7 prompts/week with 1.14× accept rate vs engineers; consulting/pro services average 68 prompts/week, highlighting cross-functional uptake.
  • Intent shift with experience: Plan-first prompts are 13.6% of first messages and rise +1.63 percentage points per standard deviation of experience, while explain-code prompts fall −0.83 pp—evidence of a semantic shift.
  • Usage intensity vs. output: Within one org, a +1σ increase in agent usage (≈459 messages/2 months) associates with +5.13 merges, +2.94K lines edited, and +0.61 more files touched per merge.

🔭 For your roadmap

  • Watch how people use agents: Log how often teams ask the agent to “plan” or “explain,” which levels approve the edits, and how long reviews take. If planning and review time shrink, you know thinking work is shifting upstream.
  • Make default-on a managed launch: Turn the agent on for everyone only after you’ve shared ready-made prompt templates, a lightweight review checklist, and a dashboard for reverts/bugfixes. Budget 6–8 weeks for the rollout to settle.
  • Teach better delegation habits: Host short show-and-tell sessions where senior engineers walk through how they brief the agent, then give juniors the same templates so they can practice and close the experience gap.
