How to tune demonstrations

      
    
        
  • Shots: number of exemplars shown before the query. Too few and the model guesses; too many and it wastes context & mixes signals.
    • 
        
  • Example fidelity: how polished each example is. Higher values mean consistent tone, structure, and correct answers; low fidelity teaches bad habits.
    • 
        
  • Coverage / diversity: how well examples span edge cases and negative cases. Higher coverage fights bias toward the majority pattern.
    • 
      

      
Adjust the sliders to see how these trade-offs influence readiness and the recommendations below.

    

  

  

    
    

      
      
      
    

    

      
Format quick guide

      
    
        
  • Balanced: lightweight scaffolding (Instruction, Output) keeps examples readable while still nudging the model toward the right structure.
    • 
        
  • Structured: explicit delimiters, slot labels, or JSON shells enforce exact formatting—ideal when downstream systems parse the response.
    • 
        
  • Minimal: raw demonstrations with almost no scaffolding. Useful to test how well the model infers patterns from pure examples, but more fragile.
